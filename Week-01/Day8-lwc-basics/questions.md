Day 8 Revision Questions

---

1. What is Lightning Web Components (LWC)?

Lightning Web Components (LWC) is Salesforce’s modern UI framework used for building fast, reusable, and scalable user interfaces.

LWC is built using:

* HTML
* JavaScript
* CSS
* XML Configuration

It follows modern web standards and supports component-based application development.

---

2. Why are reusable components useful?

Reusable components are useful because they:

* Reduce duplicate code
* Improve maintenance
* Increase development speed
* Keep UI consistent
* Simplify testing
* Improve scalability

Example:

A Header Component can be reused across multiple pages instead of creating the same UI repeatedly.

---

3. Difference between frontend and backend?

| Frontend                        | Backend                               |
| ------------------------------- | ------------------------------------- |
| Handles UI and user interaction | Handles business logic and processing |
| Displays forms and buttons      | Processes data and rules              |
| Uses HTML, CSS, JavaScript      | Uses Apex, SOQL, database services    |
| Focuses on user experience      | Focuses on system functionality       |

Frontend Example:

* Button clicks
* Form display
* Notifications

Backend Example:

* Database save
* Fee calculation
* Security validation

---

4. Why did Salesforce move toward LWC?

Salesforce introduced Lightning Web Components to provide:

* Better performance
* Faster rendering
* Modern JavaScript support
* Better scalability
* Cleaner architecture
* Easier maintenance
* Improved developer productivity

LWC improves both application speed and user experience.

---

5. Why is UI important in enterprise systems?

UI is important because users interact with applications through the user interface.

A good UI helps:

* Improve productivity
* Improve user experience
* Simplify navigation
* Reduce confusion
* Increase efficiency

Example:

A clean student dashboard helps students quickly access attendance, fees, and notifications.

---

6. Why should systems separate UI and business logic?

Separating UI and business logic improves:

* Security
* Maintainability
* Scalability
* Testing
* Code organization

Frontend handles:

* User interaction
* Display logic

Backend handles:

* Business rules
* Database operations
* Security validations

This separation makes enterprise systems easier to manage and scale.

---

7. What security risks exist in enterprise applications?

Common security risks include:

* Unauthorized access
* Data leakage
* Weak validation
* Permission issues
* Incorrect data exposure
* Broken authentication

Enterprise applications must protect sensitive data such as:

* Student records
* Financial information
* Customer details

Strong security is essential for reliable enterprise systems.

---

8. Why should developers think modularly?

Modular thinking helps developers:

* Reuse components
* Simplify development
* Improve maintenance
* Reduce bugs
* Improve scalability
* Support team collaboration

Breaking applications into smaller modules makes systems easier to understand and manage.

Additional Practice Questions

9. What are the main files inside an LWC component?

An LWC component usually contains four main files:

* HTML File → UI structure
* JavaScript File → Logic and interaction
* CSS File → Styling and design
* Meta XML File → Component configuration

Together these files create a complete Lightning Web Component.

---

10. What is component-based architecture?

Component-based architecture is a design approach where applications are divided into small reusable independent components.

Each component performs a specific responsibility.

Examples:

* Header Component
* Sidebar Component
* Dashboard Component
* Notification Component

Benefits:

* Reusability
* Easier maintenance
* Faster development
* Better scalability

---

11. What is the role of Apex in Salesforce?

Apex is Salesforce’s backend programming language.

Apex handles:

* Business logic
* Database operations
* Record processing
* Automation logic
* Security validation
* API integrations

Examples:

* Updating course seat counts
* Calculating fee balances
* Processing attendance records

Apex is mainly used for advanced enterprise logic.

---

12. Why is frontend validation important?

Frontend validation is important because it improves:

* User experience
* Faster error detection
* Input accuracy
* Reduced invalid submissions

Examples:

* Checking required fields
* Validating email format
* Preventing empty submissions

Frontend validation helps users correct mistakes before data reaches the server.

---

13. What is scalability?

Scalability means a system can grow and handle increasing:

* Users
* Data
* Transactions
* Operations

without major performance problems.

Enterprise systems must support:

* Thousands of users
* Large databases
* Continuous operations

Scalable applications remain stable as organizations grow.

14. Why are enterprise applications complex?

Enterprise applications are complex because they manage:

* Large amounts of data
* Multiple users
* Security requirements
* Integrations
* Automation
* Business rules
* Reporting systems

Examples include:

* College Management Systems
* Banking Applications
* Hospital Systems
* E-Commerce Platforms

These systems require careful architecture, security, testing, and scalability planning.
