Doubts and Discussion Questions

1. How is an App different from an Object in Salesforce?

An App acts as a workspace that combines related features, tabs, and objects for a specific business activity.
An Object is used to store data records inside Salesforce.

Example:

* A Library Management App may contain Book, Student, and Issue objects.
* The Book Object stores book-related information.

---

2. Why is configuration preferred over coding in Salesforce?

Salesforce recommends configuration first because it reduces development time and simplifies maintenance. No-code tools are easier to manage and can solve most business requirements without programming.

Benefits:

* Quick deployment
* Lower maintenance effort
* Less technical complexity
* Easier upgrades

---

3. In which situations is Apex programming required?

Apex is used when business requirements become too complex for standard configuration tools.

Examples:

* Integrating third-party applications
* Performing complex calculations
* Creating custom automation
* Processing large amounts of data

Developers use Apex to implement advanced business logic.

---

4. What is meant by multi-tenant architecture?

Multi-tenant architecture is a system where multiple customers use the same Salesforce platform infrastructure while their data remains completely separate and secure.

Advantages:

* Reduced infrastructure cost
* Automatic updates
* Better scalability
* Centralized maintenance

---

5. Why are APIs important in Salesforce?

APIs help Salesforce connect with external systems and applications.

Using APIs, organizations can:

* Exchange data between platforms
* Integrate payment systems
* Connect mobile applications
* Synchronize business information automatically

APIs improve system communication and automation.

---

6. How can developers customize Salesforce functionality?

Salesforce provides multiple development technologies for extending platform capabilities.

Developers can use:

* Apex
* Lightning Web Components
* REST and SOAP APIs
* Flows
* Custom Objects
* Triggers

These tools help build customized business solutions.

---

7. What are the different development approaches in Salesforce?

Salesforce supports two development methods.

Declarative Development

Uses built-in tools without coding.

Examples:

* Flow Builder
* Validation Rules
* Approval Processes
* Formula Fields

Programmatic Development

Uses programming for advanced customization.

Examples:

* Apex Classes
* Triggers
* Lightning Components
* API Integrations

Both approaches are combined depending on business requirements.

---

8. Compare Configuration and Coding in Salesforce

| Configuration                     | Coding                            |
| --------------------------------- | --------------------------------- |
| Uses no-code tools                | Uses programming languages        |
| Faster to implement               | Requires development effort       |
| Easier for admins                 | Mainly handled by developers      |
| Suitable for simple requirements  | Suitable for complex requirements |
| Uses standard Salesforce features | Enables advanced customization    |
| Lower maintenance                 | Higher flexibility                |

---

9. Why are Lightning Components important?

Lightning Components help developers create modern and interactive user interfaces inside Salesforce.

Advantages:

* Reusable components
* Better user experience
* Faster application performance
* Mobile-friendly design

They are commonly used in Lightning Experience applications.

---

10. What is the role of automation in Salesforce?

Automation reduces manual work and improves business efficiency.

Salesforce automation tools can:

* Send notifications automatically
* Update records
* Approve requests
* Trigger workflows
* Generate reports

This helps organizations save time and reduce errors.
